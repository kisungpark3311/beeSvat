## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

성경을 읽는 많은 분들이 "읽기는 하지만 깊이 이해하지 못하는" 상황에 놓여 있습니다. 마치 외국어로 된 아름다운 시를 번역본으로만 읽는 것과 같습니다. 번역은 대략적인 의미를 전달하지만, 원문의 미세한 뉘앙스와 구조적 아름다움은 놓치게 됩니다.

현재 성경 묵상 앱들은 크게 세 가지 한계가 있습니다:

- **너무 단순**: 번역 본문만 제공하고, 구문 구조나 원어의 의미를 알려주지 않음
- **묵상과 분석이 분리**: 묵상 앱과 원어 분석 도구가 따로 존재하여 오가며 사용해야 함
- **전문가용**: 원어 분석 도구는 신학 전공자를 위해 만들어져 일반인이 접근하기 어려움

### 1.2 목표 상황 (To-Be)

beeSvat는 "누구나 원어의 깊이로 성경을 묵상할 수 있는 안내자" 역할을 합니다. 마치 성경 원어에 능통한 목사님이 옆에 앉아서 "이 문장의 핵심 동사는 이것이고, 이런 의미가 담겨 있어요"라고 친절하게 설명해주는 것과 같은 경험을 제공합니다.

사용자는 별도의 원어 지식 없이도:

- 본문의 핵심 구조(주동사, 수식어, 접속사 관계)를 한눈에 파악
- 원어에 담긴 깊은 의미를 쉬운 해설로 이해
- 이해를 바탕으로 더 깊은 묵상으로 나아감

### 1.3 왜 지금인가? (Why Now)

- AI 기술의 발전으로 복잡한 구문 분석을 자동화할 수 있게 됨
- 주변 사람들이 성경을 더 깊이 이해하고 싶어하는 필요가 확인됨
- 기존 앱들이 이 격차를 메우지 못하고 있음
- 두란노 QT 등 일일 묵상 콘텐츠와 연동 가능한 환경이 갖춰짐

## 2. 사용자 페르소나

### 2.1 주요 페르소나

| 항목        | 내용                                             |
| ----------- | ------------------------------------------------ |
| 이름/별칭   | "묵상하는 은혜씨"                                |
| 나이/직업   | 35세 직장인, 교회 소그룹 리더                    |
| 핵심 니즈   | 매일 묵상할 때 성경 본문을 더 깊이 이해하고 싶음 |
| 불편 포인트 | 번역만으로는 아쉽고, 원어 도구는 어려워서 포기함 |
| 사용 상황   | 매일 아침 출근 전 QT, 주말 소그룹 성경 공부 준비 |

### 2.2 페르소나의 하루 (시나리오)

아침 6시 30분, 은혜씨는 커피 한 잔을 들고 거실 소파에 앉습니다. 스마트폰으로 beeSvat 앱을 열면 오늘의 QT 본문(두란노 매일성경)이 자동으로 표시됩니다.

본문을 읽으면서 "구문 분석" 버튼을 누르면, 본문의 핵심 동사가 강조되고 문장의 구조가 시각적으로 펼쳐집니다. "이 구절의 핵심 동사는 '사랑하다(아가파오)'인데, 이것은 무조건적 사랑을 의미하는 단어입니다"라는 해설이 함께 나옵니다.

은혜씨는 "아, 이 문장이 이런 구조였구나" 하며 새로운 깨달음을 얻고, 묵상 노트에 오늘의 묵상을 기록합니다. 주말에는 이 분석 결과를 참고하여 소그룹 성경 공부를 인도합니다.

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (온보딩/로그인)

- [ ] 사용자로서, 이메일 또는 소셜 계정으로 간편하게 가입할 수 있어야 한다. 왜냐하면 복잡한 가입 절차는 묵상의 흐름을 끊기 때문이다.
- [ ] 사용자로서, 로그인 상태가 유지되어야 한다. 왜냐하면 매일 아침 빠르게 묵상을 시작해야 하기 때문이다.
- [ ] 사용자로서, 간단한 온보딩을 통해 앱 사용법을 익힐 수 있어야 한다. 왜냐하면 구문 분석이 생소한 개념이기 때문이다.

### FEAT-1: 성경 구문 분석 (MVP)

- [ ] 묵상자로서, 성경 본문의 주동사와 수식어를 자동으로 구분해서 볼 수 있어야 한다. 왜냐하면 문장의 핵심 의미를 파악하기 위함이다.
- [ ] 묵상자로서, 구문 분석 결과에 대한 쉬운 해설을 읽을 수 있어야 한다. 왜냐하면 원어 지식 없이도 의미를 이해해야 하기 때문이다.
- [ ] 묵상자로서, 구문 구조를 시각적으로 볼 수 있어야 한다. 왜냐하면 문장의 전체 구조를 한눈에 파악하고 싶기 때문이다.

### FEAT-2: 원어(히브리어/헬라어) 동사 분석 (v2 이후)

- [ ] 묵상자로서, 각 동사의 원어 형태와 시제/태/법을 확인할 수 있어야 한다. 왜냐하면 원어의 미묘한 뉘앙스를 알고 싶기 때문이다.
- [ ] 묵상자로서, 동일 원어가 사용된 다른 구절을 참조할 수 있어야 한다. 왜냐하면 성경 전체 맥락에서 단어의 의미를 이해하고 싶기 때문이다.

### FEAT-3: 묵상 노트 자동 생성 (v2 이후)

- [ ] 묵상자로서, 구문 분석 결과를 바탕으로 묵상 노트 초안을 자동 생성할 수 있어야 한다. 왜냐하면 분석에서 묵상으로의 전환을 자연스럽게 하고 싶기 때문이다.
- [ ] 묵상자로서, 자동 생성된 노트를 편집하고 저장할 수 있어야 한다. 왜냐하면 개인적인 깨달음을 추가하고 싶기 때문이다.

### FEAT-4: 성경 본문 자동 가져오기 (v2 이후)

- [ ] 묵상자로서, 두란노 QT 오늘의 본문이 자동으로 표시되어야 한다. 왜냐하면 별도로 본문을 찾는 수고를 덜고 싶기 때문이다.
- [ ] 묵상자로서, 원하는 성경 구절을 직접 검색하여 분석할 수 있어야 한다. 왜냐하면 QT 본문 외에도 궁금한 구절이 있기 때문이다.

## 5. 범위 관리

### 5.1 MVP (반드시 포함)

- FEAT-0: 기본 온보딩/로그인 (이메일 가입, 소셜 로그인)
- FEAT-1: 성경 구문 분석 (주동사/수식어 자동 파싱 & 해설)

### 5.2 옵션 (시간 허용 시)

- FEAT-4의 일부: 성경 구절 직접 입력/검색 기능
- FEAT-3의 일부: 간단한 묵상 메모 저장

### 5.3 Non-goals (명시적 제외)

- 원어 사전 기능: 전문 사전은 기존 도구가 잘 제공하고 있으므로 중복 투자하지 않음
- 소셜 기능: 커뮤니티/공유 기능은 MVP에서 핵심 가치와 거리가 있음
- 결제 시스템: 기부/후원 기반 수익 모델은 사용자 기반 확보 후 도입
- 오프라인 모드: 초기에는 온라인 환경에서만 지원

## 7. 해결 가설과 실험

### 7.1 가설

> 만약 성경 본문의 구문 구조를 자동으로 분석하여 주동사와 수식어를 시각적으로 보여주고 쉬운 해설을 제공하면, 원어 지식이 없는 묵상자도 더 깊은 이해를 경험할 것이다. 왜냐하면 구문 구조의 파악은 본문 해석의 첫 번째 단계이며, 이것이 자동화되면 진입 장벽이 크게 낮아지기 때문이다.

### 7.2 실험 루프

```
실험 -> 관측 -> 학습 -> 다음 가설
```

| 단계 | 내용                                                                   |
| ---- | ---------------------------------------------------------------------- |
| 실험 | FEAT-1(구문 분석) MVP를 5명의 묵상자에게 1주일간 사용하게 함           |
| 관측 | 일일 사용 빈도, 분석 결과 만족도(별점), 정성 피드백 수집               |
| 학습 | 만족도 4.0점 이상이면 성공, 미만이면 해설 방식 재검토                  |
| 다음 | 성공 시 FEAT-2(원어 동사 분석) 추가, 실패 시 해설 UI/UX 개선 후 재실험 |
